The `@quranjs/api` library provides a comprehensive TypeScript SDK for interacting with the Quran.com API. It offers a clean, strongly-typed interface for accessing Quranic content, including verses, chapters, audio recitations, translations, and more.

## Library Structure

The library is built around the main `QuranClient` class, which provides access to different API modules:

```ts
import { QuranClient } from "@quranjs/api";

const client = new QuranClient({
  clientId: "your-client-id",
  clientSecret: "your-client-secret"
});

// Access different API modules
client.chapters   // Chapter-related operations
client.verses     // Verse-related operations
client.audio      // Audio recitation operations
client.search     // Search functionality
client.resources  // Resource metadata (translations, tafsirs, etc.)
client.juzs       // Juz-related operations
```

## API Modules

### Chapters API

Access chapter (surah) information, including metadata and chapter details.

```ts
// Get all chapters
const chapters = await client.chapters.findAll();

// Get specific chapter
const alFatiha = await client.chapters.findById("1");

// Get chapter info with detailed description
const chapterInfo = await client.chapters.findInfoById("1");
```

### Verses API

Retrieve verses (ayahs) by various criteria with support for translations, tafsirs, and audio.

```ts
// Get specific verse
const ayahAlKursi = await client.verses.findByKey("2:255");

// Get all verses in a chapter
const verses = await client.verses.findByChapter("1");

// Get verses by page, juz, hizb, or rub
const pageVerses = await client.verses.findByPage("1");
const juzVerses = await client.verses.findByJuz("1");
```

### Audio API

Access audio recitations for chapters and verses from different reciters.

```ts
// Get chapter recitations for a reciter
const recitations = await client.audio.findAllChapterRecitations("2");

// Get verse recitations by chapter
const verseAudio = await client.audio.findVerseRecitationsByChapter("1", "2");
```

### Search API

Search through Quranic content with advanced filtering options.

```ts
// Basic search
const results = await client.search.search("light");

// Search with options
const arabicResults = await client.search.search("نور", {
  language: Language.ARABIC,
  size: 10
});
```

### Resources API

Access metadata about available translations, tafsirs, reciters, and other resources.

```ts
// Get available translations
const translations = await client.resources.findAllTranslations();

// Get available reciters
const reciters = await client.resources.findAllChapterReciters();

// Get tafsir information
const tafsirs = await client.resources.findAllTafsirs();
```

### Juzs API

Access information about Juzs (also known as Para or Siparah).

```ts
// Get all juzs
const juzs = await client.juzs.findAll();
```

## Key Features

### Strong TypeScript Support

The library provides comprehensive TypeScript definitions for all API responses, request parameters, and configuration options.

### Authentication Handling

Automatic OAuth2 token management with caching and renewal.

### Flexible Configuration

Support for custom base URLs, default parameters, and custom fetch implementations.

### Error Handling

Comprehensive error handling with meaningful error messages.

### Validation

Built-in validation for chapter IDs, verse keys, and other parameters.

## Core Types

The library exports many TypeScript interfaces for working with Quranic data:

- `Chapter` - Chapter (surah) information
- `Verse` - Verse (ayah) data with optional translations and audio
- `Juz` - Juz information
- `SearchResult` - Search result data
- `RecitationResource` - Audio recitation metadata
- `TranslationResource` - Translation metadata
- `TafsirResource` - Tafsir (commentary) metadata
